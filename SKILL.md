# Idea For Sale

Read `references/workflow.md` first.

如果输入以人物、传记、演讲、访谈、书信、论文或思想体系为主，再读 `references/person-distillation.md`。

开始执行前，再读 `references/material-collection.md`。

这个 skill 不负责追热点本身，而负责把多源素材加工成：

- 值不值得做的二层洞察
- 能不能卖的产品机会
- 能不能在 7 天内验证
- 能不能沉淀成复利资产

这里的一层素材包括：

- 热点新闻
- 产品发布
- 博主观点
- 用户讨论
- 名人传记、演讲、访谈、书信、论文、决策记录

优先目标不是“写出一篇好分析”，而是筛出“更可能卖得动、做得出、还能长出来”的方向。

## 什么时候使用

当用户在做下面这些事时使用：

- 想从 AI 热点、新闻、产品发布、行业讨论里找可卖方向
- 想把“有意思的话题”转成“能成交的产品机会”
- 想判断某个 idea 是否适合咸鱼单卖、私域成交、咨询、服务或模板化
- 想快速验证某个方向是否有人愿意花时间、花钱
- 想做多轮圆桌讨论，把方向连续迭代再收敛
- 想把乔布斯、达尔文等人物的思想、方法论、判断框架蒸馏成可卖产品

如果用户只是要新闻摘要、人物介绍、普通深度研究、文章写作，而不关心产品机会、成交或验证，不要用这个 skill。

## 核心原则

1. 不做闭门分析
   没有多源素材，不进入圆桌讨论。用户提供的初始材料只算起点，不算充分输入。

2. 不停留在一层信息
   素材本身不是结论，要继续追问被忽略的约束、二级后果、迁移机制和真实痛点。

3. 先看差异，再谈产品化
   每个热点都必须先回答：它和现有工具、旧工作流、相邻产品到底哪里不同；它的突破是不是足以让用户改变行为。

4. 不用“有趣”代替“可卖”
   一个方向如果不能说明谁会买、为什么现在买、买完得到什么，就不能进入主推方向。

5. 不用“高复利”掩盖“卖不动”
   如果 `Pain < 60` 或 `WTP < 60`，即使 `Compounding` 很高，也不能进入最终主推方向。

6. 圆桌必须带新信息
   每一轮碰撞都必须引入至少一条此前没有的新信息。没有新信息，这一轮不算有效轮次。

7. 发散阶段优先使用 `brainstorming`
   如果环境里存在 `brainstorming` skill，创意碰撞与发散轮阶段必须优先使用它来生成、扩展、改写和重组方向；只有在不可用时，才退回普通创意碰撞。

8. 正式圆桌优先使用 `ljg-roundtable`
   如果环境里存在 `ljg-roundtable` skill，正式圆桌阶段必须优先使用它；只有在不可用时，才退回普通结构化辩论。在本 skill 中，`ljg-roundtable` 作为嵌入式子流程使用：每轮圆桌完成后自动提炼结论并进入下一阶段，不等待用户用 `可 / 止 / 深入此节 / 引入新人物` 手动推进。

9. miss 后必须切换目标
   如果某一轮试图解决的问题没有拿到新信息，下一轮不得继续盯着同一个问题。

10. 不做无证据自嗨
   每个关键判断都要标出证据强度，并用证据等级修正总分。

11. 到了可收敛状态就提前停
   满足最少 3 轮后，如果主推方向已经稳定领先、关键风险已被压缩、验证动作足够清晰，就不必机械跑到第 10 轮。

## 两阶段流程

```text
Phase 1: BASELINE
  收集素材 -> 做初版洞察 -> 形成 baseline 候选方向

Phase 2: IMPROVE
  Round 1 先扫描所有缺口 -> 选择最弱项 ->
  补新信息 -> 重新碰撞 -> 重新打分 -> keep / miss ->
  miss 后强制切换 -> 直到收敛或达到 10 轮
```

## 运行时状态文件

开始时创建 `idea-state.md`，每轮都更新。这个文件是运行时产物，不属于 skill 定义本身。

至少记录：

```text
phase: BASELINE | IMPROVE
round: 0
budget: 10
current_target: none
best_direction: none

## source inventory
- bucket: topic/event/person
- bucket: user voice
- bucket: competitor/substitute
- bucket: adjacent tools/old workflow
- bucket: transaction signal
- bucket: counterexample

## round log
- round: 1
  target: "缺少真实用户痛点原话"
  oracle_call: "补 5 条用户原话"
  new_info: "发现 3 个高频抱怨"
  action: keep
  next_constraint: none
```

## 默认执行顺序

1. 素材收集
2. baseline 压缩
3. baseline 洞察
4. 差异与突破扫描
5. baseline 可迁移思想
6. baseline 创意碰撞
7. Round 1 全面扫描缺口
8. 选择最弱项并补新信息
9. 回到圆桌重新碰撞、重排、重打分
10. 满足最少 3 轮后判断是否收敛
11. 最多 10 轮，超出则强制收敛

每轮迭代都必须明确回答：

- 这轮新增了什么外部信息
- 这个方向的购买者是谁
- 他们为什么现在会在意
- 买完能得到什么
- 最快成交路径是什么
- 这个方向怎样沉淀复利资产
- 上一轮的最大弱点是否被解决

## 评分与约束

先读 [references/scoring-rubric.md](references/scoring-rubric.md)。

评估结构固定为 4 层：

- `Gate`
  硬门槛。不满足则降权或淘汰。
- `Score`
  主评分。用于比较候选方向。
- `Penalty`
  惩罚项。用于扣减看起来好、实则难卖或难交付的方向。
- `Evidence`
  证据等级。用于把主观高分修正为可信高分。

默认优先顺序：

- 先看 `Gate`
- 再看 `adjusted_score`
- 再看 `Pain`、`WTP`、`Compounding` 的结构是否健康
- 最后才看 `Novelty`

如果这轮没有新增外部信息，只允许更新素材缺口，不允许提升方向排名。

## 执行要求

- 不要跳过素材收集直接开始圆桌
- 不要跳过脑暴直接进入正式圆桌；先扩张方向，再做冲突裁剪
- 不要只描述“它很强”，必须说清它和什么不一样、为什么现在才值得看
- 不要只给泛泛方向，至少给出具体产品形式
- 不要只说“可做咨询/课程/社群”，必须说明具体卖什么
- 不要省略定价和渠道判断，哪怕只是区间和优先级
- 不要把创意碰撞做成一层排列组合，必须制造真实张力和互斥选择
- 不要把“复利”写成抽象口号，必须落到资产沉淀
- 不要让圆桌停留在主观争论，每轮必须补新信息再回来碰撞
- 不要让最终结论脱离量化结果

## 默认产品方向清单

圆桌阶段至少覆盖这些方向，再决定是否淘汰：

- 模板/提示词/工作流包
- 咨询诊断
- 陪跑/训练营
- 代做服务
- 信息差单卖品
- 小型工具或自动化助手
- 情报订阅或案例拆解

如果用户已经限定赛道、用户群体或交付形式，只围绕限定范围讨论。
