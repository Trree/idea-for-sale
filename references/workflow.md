# Workflow

这个 workflow 的目标是把热点信息或人物思想推进到“可卖机会”，而不是停在分析层。

它借用了 `deepresearch` 的 5 个骨架：

- 先做 baseline，再继续改进
- Round 1 先全面扫描，再挑最弱项
- 每轮必须引入新信息
- miss 后必须切换目标
- 用状态文件记录轮次、目标、证据和限制

## 输入类型

这个 workflow 支持两大类输入：

- `事件型输入`
  热点、新闻、产品发布、社媒讨论、用户抱怨。
- `人物型输入`
  名人传记、演讲、访谈、书信、论文、公开决策记录、思想体系。

如果是人物型输入，先结合 [person-distillation.md](person-distillation.md) 把人物蒸馏成：

- 核心母题
- 判断框架
- 决策启发式
- 可迁移方法
- 不适用边界

然后再进入后续产品化流程。

## Phase 1: BASELINE

### 第 1 步：素材收集

先读 [material-collection.md](material-collection.md)。

开始圆桌前，至少覆盖下面 6 个桶中的 5 个：

- `topic/event/person`
- `user voice`
- `competitor/substitute`
- `adjacent tools/old workflow`
- `transaction signal`
- `counterexample`

如果少于 5 个桶，先补素材，不进入后续步骤。

### 第 2 步：压缩一层信息或人物母题

把输入压成一句话：

- 发生了什么
- 谁在关注
- 市场上已经有哪些一层解读

如果是人物型输入，改成：

- 这个人物最值得蒸馏的核心命题是什么
- 公众通常如何误读他
- 这套思想最可能服务哪类用户

要求：

- 不要急着下结论
- 不要堆背景
- 只保留后续判断需要的最小上下文

### 第 3 步：提炼 baseline 二层洞察

至少回答下面 4 个问题：

- 大多数人忽略了什么
- 真正变化的是成本、能力、分发、组织，还是用户行为
- 如果这件事持续 3 个月，会出现什么二级后果
- 哪些旧需求会被重新激活

输出 3 到 5 条二层洞察，不要超过。

如果是人物型输入，这里的二层洞察优先回答：

- 这个人物真正有价值的不是观点，而是哪套思维动作
- 哪些被大众神话的部分其实不适合产品化
- 哪些部分能转成模板、训练、诊断、陪跑或情报产品

### 第 4 步：差异与突破扫描

在进入可迁移思想和脑暴前，先做一轮差异扫描。

至少回答：

- 这个热点和当前最接近的 2-3 个工具相比，真正不同在哪里
- 这个不同是“更好一点”，还是“改变工作流”
- 它的突破点落在：
  `质量` / `速度` / `成本` / `控制力` / `多语言` / `编辑一致性` / `规模化`
  中的哪一项
- 哪些地方看起来像突破，其实只是演示更强
- 哪些用户会因为这个差异而改变原来的工具选择

如果差异不能触发换工作流，就不要高估它的商业价值。

### 第 5 步：抽取 baseline 可迁移思想

把事件级信息抽成方法级信息：

- 背后依赖的结构性变化是什么
- 这个变化能迁移到哪些相邻场景
- 哪些地方会重复出现同类需求

这一步的目标是找“不是只靠一个热点活着”的方向。

如果是人物型输入，这一步必须额外产出：

- `人物的核心问题意识`
- `人物常用的判断标准`
- `人物的反直觉动作`
- `哪些场景适合迁移`
- `哪些场景不适合迁移`

### 第 6 步：baseline 创意碰撞

不要直接从洞察跳到 5 个成品方向。先做碰撞矩阵。

如果环境里存在 `brainstorming` skill，这一步必须优先使用它作为发散引擎，而不是直接手写一轮静态矩阵。

在本 workflow 中，`brainstorming` 不是独立设计流程，而是嵌入式脑暴器：

- 先根据当前素材提出尽可能多的方向变体
- 再对这些变体做重组、反转、极端化和打包
- 最后才进入筛选与正式圆桌

不要在这一步过早追求结论。这里的目标是“长更多方向”，不是“马上选对方向”。

基础轴：

`素材 x 人群 x 痛点 x 能力杠杆 x 产品形式 x 交付轻重 x 渠道 x 复利资产`

然后再加 3 个强制碰撞轴：

- `贵但强` vs `便宜但快`
- `卖结果` vs `卖工具` vs `卖判断`
- `高客单低频` vs `低客单高频`

再补 2 个反向碰撞：

- `这个方向为什么会卖不动`
- `什么替代品会直接杀死它`

至少先产出：

- `20-40` 个粗方向
- 每个粗方向只写一句话价值
- 标记哪些方向明显重复、明显虚、明显太重
- 标记哪些方向彼此互斥、哪些方向可以打包

如果使用 `brainstorming`，额外要求至少做 4 种发散动作中的 2 种：

- `改人群`
  同一个能力，换不同买家
- `改结果`
  同一个素材，换不同交付结果
- `改形态`
  从服务改成模板、从模板改成诊断、从诊断改成订阅
- `极端化`
  做成“最便宜”“最快”“最重”“最贵”“最窄”“最反常识”的版本

无论是否使用 `brainstorming`，都要额外做一次：

- `最小痛点解法`
  把每个强方向压缩成“最小可交付、最小可理解、最小可验证”的版本

然后再筛到 `5-7` 个进入正式圆桌。

## Phase 2: IMPROVE

### 第 7 步：Round 1 全面扫描缺口

先扫描当前 deliverable 的全部缺口，再选最弱项，不要直接挑自己最喜欢的话题。

默认扫描维度：

- `source coverage`
  哪个素材桶仍然太薄
- `user evidence`
  有没有真实用户原话和购买动机
- `competitive reality`
  有没有竞品、替代品、免费替代路径
- `differentiation`
  和相邻工具相比，突破点到底清不清楚
- `transaction signal`
  有没有价格带、成交页面、预售、需求信号
- `boundary`
  有没有不适用边界和失败案例
- `direction diversity`
  方向是不是过于同质
- `delivery clarity`
  买完得到什么是否说清

按缺口从大到小排序，选出 `current_target`。

### 第 8 步：补新信息

针对 `current_target` 做一个聚焦的外部信息动作。

有效新信息包括：

- 新的用户原话
- 新的竞品页、定价页、销售页
- 新的相邻工具比较案例
- 新的失败案例或反例
- 新的行业映射案例
- 新的成交信号
- 新的边界条件

如果这一步没有拿到 materially new information，记为 `miss`。

### 第 9 步：正式圆桌讨论

圆桌的输入必须包含本轮新信息。

如果环境里存在 `ljg-roundtable` skill，这一步必须优先使用它来执行正式圆桌。

但在本 workflow 中，`ljg-roundtable` 不是独立终局，而是嵌入式分析器：

- 完成 1 次定义性开场
- 完成 1 轮到 2 轮高张力交锋
- 主持人提炼核心裂缝
- 自动进入量化评估与下一轮扫描

不要在这里停下来等待用户选择 `可 / 止 / 深入此节 / 引入新人物`。默认自动推进，除非用户明确要求停下。

圆桌议题不要用宽泛问题，例如：

- 错误：`Kimi K2.6 能不能产品化？`
- 正确：`围绕 Kimi K2.6，这一轮最值得卖的是“评测迁移包”“接入配置包”还是“私有部署服务”？`

圆桌角色选择不能只有支持者，至少要形成张力网络。默认优先包含 5 类角色中的 3-5 类：

- `买方`
  真正掏钱的人，例如 CTO、独立开发者、运营负责人。
- `卖方`
  真正能交付的人，例如顾问、开发者、服务商。
- `怀疑者`
  专门攻击“这东西为什么卖不动”的角色。
- `替代品代表`
  代表现有免费方案、闭源竞品或人工流程。
- `资产配置者`
  关注复利、规模化、长期护城河的人。

如果使用 `ljg-roundtable`，要把议题压成“最尖锐的交易冲突”，并要求输出至少：

- `参会角色与立场`
- `定义性问题`
- `2-4` 轮高张力交锋
- `主持人提炼的核心裂缝`
- `哪些方向被圆桌直接淘汰`
- `哪些方向因新冲突被提升`
- `圆桌结束后自动进入的下一阶段`

至少提出 5 个方向。每个方向都必须写清：

- `方向名称`
- `目标人群`
- `一句话价值`
- `产品形式`
- `建议价格带`
- `优先渠道`
- `预期交付`

圆桌不是列清单，而是辩论：

- 为什么这个方向可能卖得动
- 为什么它也可能卖不动
- 它最大的短板是什么
- 本轮新增信息改变了什么
- 它和现有工具相比最关键的差异是什么
- 这个差异为什么足以让用户切换工作流
- 如果把它压成一个小而美的 MVP，最小版本是什么
- 哪个角色最强烈反对它
- 它输给替代品的最短路径是什么
- 如果只能卖 1 个 SKU，为什么不是它

如果本轮没有新增信息，不允许提升任何方向排名。

### 第 10 步：量化评估

按评分标准给每个方向打分，并记录：

- `weighted_score`
- `penalty_score`
- `confidence_score`
- `adjusted_score`
- `本轮最大风险`

### 第 11 步：验证动作设计

每个方向至少指定 1 个 7 天内能执行的最小验证动作，例如：

- 咸鱼标题 + 定价 + 交付承诺
- 私聊成交话术
- 群内测试文案
- 简单预约页
- 样品描述页
- 预售招募文案

同时补一句：

- `最小可卖版本`
  如果只能做一个小而美版本，现在具体卖什么

### 第 12 步：判断 keep 或 miss

如果本轮新增信息改变了排序、增强了证据、修复了关键低分维度：

- `action = keep`
- 更新 `best_direction`
- 进入下一轮扫描

如果本轮没有拿到 materially new information：

- `action = miss`
- 明确写出缺口
- 下一轮不得继续盯着同一个 `current_target`

### 第 13 步：进入下一轮

从第 2 轮开始，不是重写，而是带着问题改造：

- 用户换不换
- 产品形式换不换
- 价格带换不换
- 渠道换不换
- 交付是不是要更轻
- 和哪个旧工具/旧工作流比较
- 哪个素材桶还要补
- 上一轮 miss 的目标本轮是否必须避开

目标是修复上一轮里分数最低的关键维度。

## 轮次预算

第 3 轮开始，进一步压缩为最小可卖单元：

- 不再讨论“大而全产品”
- 优先讨论一周内能卖第一单的版本
- 优先讨论能沉淀资产的成交方式

默认至少跑 3 轮；如果方向之间仍然接近、关键风险未解决、或验证动作仍然模糊，可以继续迭代，但最多 10 轮。

## 最终收敛

只输出 3 类结果：

- `主推方向`
  `adjusted_score` 高，且关键指标过线。
- `备选方向`
  总体可行，但还有关键短板未解决。
- `淘汰方向`
  连续两轮弱，或硬门槛不过。

如果已经跑到第 10 轮，无论是否满意，都必须基于当前最优证据强制收敛，不再继续发散。

## 收敛后必须补的最终整理

进入最终收敛后，不要立刻结束。还必须补两步：

### 最终整理 1：结构化输出

把最终结论整理成稳定结构，至少包含：

- 热点
- 一句话判断
- 最值得先做的方向
- 为什么选它
- 不优先的方向
- 最小可卖版本
- 解决的单点痛点
- 目标用户
- 交付内容
- 建议定价
- 验证标准

### 最终整理 2：具体操作指南

把“应该做什么”展开成可以执行的步骤，至少包含：

- 第 1 步做什么
- 第 2 步做什么
- 第 3 步做什么
- 先联系谁
- 用什么文案测试
- 收到什么反馈后迭代

### 最终整理 3：落盘到 Markdown

把最终结果写入当前工作目录下的本地 Markdown 文档。

要求：

- 文件必须是 `.md`
- 文档内容至少包含 `结构化输出` 和 `具体操作指南` 两部分
- 文件名要和主题直接相关，便于后续继续迭代

## 提前停止判据

满足最少 3 轮后，如果下面条件同时大体成立，可以提前停止，不必继续迭代到第 10 轮：

- `主推方向稳定领先`
  连续 2 轮 `adjusted_score` 排名第一，且领先第二名至少 `10` 分。
- `关键保护规则通过`
  `Pain >= 60`、`WTP >= 60`，并且所有 Gate 通过。
- `验证动作足够清晰`
  已经能明确写出验证对象、渠道、文案/标题、报价、交付承诺、成功信号、失败信号。
- `本轮修改收益变小`
  相对上一轮，主推方向的 `adjusted_score` 提升小于 `5` 分，且没有新增强证据改变排序。
- `最大风险已知且可接受`
  剩余风险已经从“方向不成立”收缩到“执行细节待验证”。

如果只满足其中一部分，不要提前停止，继续迭代。

## 强制收敛规则

一个方向不能进入 `主推方向`，如果：

- `Pain < 60`
- `WTP < 60`
- 7 天内无法验证
- 无法一句话说清“买完得到什么”

## 输出纪律

- 每轮都要写明新增了什么信息，以及来自哪个素材桶
- 每轮都要说明相对上一轮改了什么
- 每轮都要记录为什么某些方向上升或下降
- miss 时要显式写出“本轮没有拿到新信息”
- 如果所有方向都不满足门槛，要明确说“当前无主推方向”
- 最终必须产出可落盘的 Markdown 版结果，而不是只在对话里结束

## 每轮结束时的标准决策句式

每轮结束后，必须用固定句式收口，避免只给印象式总结。

按下面顺序输出：

### 1. 本轮结论

`本轮结论：当前领先方向是「X」，因为它在 Pain / WTP / Compounding 上的结构最稳，当前 adjusted_score 为 Y。本轮新增信息是 ______。`

如果当前没有合格主推方向，用：

`本轮结论：当前无合格主推方向，原因是关键 Gate 未通过，或 Pain / WTP 未过线。本轮新增信息是 ______。`

### 2. 排名变化

`排名变化：方向 A 上升 / 下降到第 N 名，主要因为 ______。`

至少写出：

- 前 2 名为什么排在前面
- 至少 1 个下滑方向为什么下滑

### 3. 关键修复

`关键修复：本轮主要修复了「______」这个低分维度，方法是把方向从「旧版本」改成「新版本」。`

如果没有真正修复，要明确写：

`关键修复：本轮未有效修复关键短板，当前仍卡在 ______。`

如果是 miss，用：

`关键修复：本轮 miss。围绕「______」没有拿到新信息，下一轮必须切换目标。`

### 4. 保留还是淘汰

对每个方向只允许用下面 4 种状态之一：

- `继续放大`
  `继续放大：这个方向通过 Gate，且本轮分数提升明显，值得下一轮继续加码。`
- `继续观察`
  `继续观察：这个方向暂未淘汰，但关键证据仍不足，需要下一轮补证据。`
- `降级处理`
  `降级处理：这个方向仍有价值，但更适合作为备选、低价版或更轻交付形态。`
- `直接淘汰`
  `直接淘汰：这个方向本轮触发了 ______，不再继续投入。`

### 5. 下一轮唯一重点

`下一轮重点：只解决「______」这一件事，因为它是当前最大的排序阻力。`

这里必须只写一个重点，不允许列多个。

### 6. 是否收敛

如果未达到提前停止条件，用：

`是否收敛：否。原因是 ______，因此进入第 N+1 轮。`

如果达到提前停止条件，用：

`是否收敛：是。原因是主推方向已连续领先、保护规则通过、验证动作清晰，进入最终收敛。`
