# Material Collection

当前 skill 的第一责任不是分析，而是把输入补成“可碰撞的素材池”。

## 素材桶

默认收集 6 个桶，至少命中 5 个桶再进入正式圆桌：

### 1. topic / event / person

记录这件事本身：

- 热点、产品发布、政策变化、人物思想、方法论
- 一层市场叙事是什么
- 为什么它现在值得看

### 2. user voice

记录真实用户语言，而不是 agent 转述。

优先收集：

- 用户抱怨
- 用户提问
- 用户想买但不会选
- 用户说“太复杂”“太贵”“太慢”“不会用”

目标是找到真实痛点原话。

### 3. competitor / substitute

记录用户当前已经怎么解决。

包括：

- 直接竞品
- 免费替代品
- 通用工具替代
- 人工替代方式

如果不清楚替代品，就很难判断有没有付费空间。

### 4. adjacent tools / old workflow

记录这个热点与现有工具或旧工作流相比，到底变了什么。

包括：

- 当前最接近的同类工具
- 用户过去的做法
- 新旧路径在速度、质量、成本、控制感、可编辑性上的差异
- 哪一步是突破点，哪一步仍然没变

如果说不清“和什么相比更好”，就很难判断这是不是突破。

### 5. transaction signal

记录任何接近成交的信号：

- 定价页
- 销售页
- 咸鱼、淘宝、知识星球、社群招募
- 预约、预售、付费咨询
- 用户明确表达愿意花钱

这类素材决定 `WTP` 和 `Speed` 的可信度。

### 6. counterexample

记录失败案例、边界案例和反例：

- 已经做过但没卖动的东西
- 用户看似感兴趣但不付钱
- 方向过重、过虚、过窄的证据

没有 counterexample，圆桌容易自嗨。

## 素材收集的最低产物

进入 baseline 前，至少给出：

- 每个素材桶 1 到 3 条材料
- 哪些桶已经足够
- 哪些桶仍然缺
- 当前最危险的素材缺口
- 当前最重要的“差异比较对象”

## 创意碰撞前的硬规则

如果缺下面任一项，不要进入正式圆桌：

- 至少 3 条用户语言
- 至少 2 个替代方案
- 至少 2 个相邻工具或旧工作流比较点
- 至少 1 个成交或定价信号
- 至少 1 个反例或失败案例

如果拿不到，不要假装完整，必须在输出里明确标记缺口。
